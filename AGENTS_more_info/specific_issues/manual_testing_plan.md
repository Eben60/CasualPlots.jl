# Comprehensive Manual Testing Plan for CasualPlots.jl

This plan covers an extensive manual testing sequence for `CasualPlots.jl`. It uses the environment and dummy variables generated by `src/scripts/casualplots_test.jl` and `src/scripts/casualplots_test_setup.jl`.

## Prerequisites
1. Open a Julia REPL (`v1.12`).
2. Activate the package environment.
3. Ensure the file reading packages are loaded *before* starting the GUI (this triggers the package extensions). Also load the helper Package:
   ```julia
   using ShareAdd; @usingany CSV, XLSX, AgenticTesting
   ```
4. Run the test script:
   ```julia
   include("src/scripts/casualplots_test.jl")
   ```
   This will start the Bonito/Electron app and populate the `Main` environment with the `caspl_*` test variables (arrays and DataFrames).

## Phase 1: Arrays Source Mode (X, Y)
**Goal:** Verify basic Array/Matrix handling, unitful unification, congruence filtering, and range selection.

1. **Basic Plotting:**
   - In the **Source** tab, under "X, Y Arrays" mode.
   - Select **X Variable**: `caspl_x_10`.
   - *Expected:* The **Y Variable** dropdown updates to show only congruent variables (length-10 arrays/matrices). Incompatible-length arrays (e.g. `caspl_x_100`) and 3D tensors (`caspl_3d`) must NOT appear.
   - Select **Y Variable**: `caspl_x_to_3by2`.
   - Click **(Re-)Plot**.
   - *Expected:* The plot appears with default labels. The data table populates correctly.
2. **Matrix as Y Variable:**
   - Select **X Variable**: `caspl_x_10`, **Y Variable**: `caspl_ys10` (a 10×2 matrix).
   - Click **(Re-)Plot**.
   - *Expected:* Both columns of the matrix are plotted as separate Y series. Legend reflects two series.
3. **Range Selection:**
   - Modify `range_from` to `2` and `range_to` to `8`.
   - Click **(Re-)Plot**.
   - *Expected:* Plot shows only the subset (7 points). The data table shows only the filtered rows.
4. **Unitful Arrays — axis label:**
   - Select **X Variable**: `caspl_x_10`, **Y Variable**: `caspl_u_10` (values in `mm²`).
   - Click **(Re-)Plot**.
   - *Expected:* The plot renders. The Y-axis label includes the unit string (e.g. `mm²`).
5. **REPL Export Check:**
   - After a successful plot, in the REPL verify:
     ```julia
     cp_figure     # should be a Makie Figure
     cp_figure_ax  # should be an Axis object
     cp_figure_ax.title = "REPL Test Title"  # change should appear live in the plot
     ```
   - *Expected:* Both globals are accessible and modifying `cp_figure_ax` immediately updates the displayed plot.

## Phase 2: DataFrame Source Mode & Normalization
**Goal:** Verify DataFrame column selection, data cleansing, and normalization rules.

1. **Simple DataFrame — sanity check:**
   - Switch the data source radio button to **DataFrame**.
   - Select `caspl_df_simple` from the dropdown.
   - Select columns `x` (X-axis), `y1`, `y2`.
   - Click **(Re-)Plot**.
   - *Expected:* A clean two-series plot with a correct table view.
2. **Large DataFrame with Multiple Y Columns:**
   - Select `caspl_df_exp` from the dropdown.
   - Check columns `x`, `y2`, `y4`.
   - Click **(Re-)Plot**.
   - *Expected:* Plot shows multiple Y series. Table shows selected columns.
3. **Unitful DataFrame Columns — Incompatible Dimensions:**
   - Select `caspl_df_unitful` from the dropdown.
   - Select `index` (X-axis), then both `area` (in `mm²`) and `linear` (in `mm`) as Y columns.
   - Click **(Re-)Plot**.
   - *Expected:* A warning popup indicating that incompatible units were stripped, allowing plotting on a common (unitless) axis.
4. **Data Cleansing — Compatible Unit Unification:**
   - Select `caspl_df_unitmix` from the dropdown.
   - Select `index` (X-axis) and `areacm`, `areammcm` as Y columns (compatible area units, different scales).
   - Click **(Re-)Plot**.
   - *Expected:* Units are silently unified; the plot renders without any warning.
5. **Data Cleansing — Missing/Non-Numeric Values (<10%):**
   - Still using `caspl_df_unitmix`, select `index` (X-axis) and `unimiss` as a Y column (contains one `"Missis"` string among 25 rows).
   - Click **(Re-)Plot**.
   - *Expected:* A warning modal states that non-numeric values were replaced with `missing`. The remaining data plots correctly.

## Phase 3: Plot Formatting
**Goal:** Ensure user customizations persist across replot events and that all format options work correctly.

1. **Labels & Axis Limits:**
   - Start from a freshly plotted chart (e.g. `caspl_df_simple`).
   - Go to the **Format** tab.
   - Set a custom `Title`, `X-Label`, and `Y-Label`.
   - Set explicit `X-min`, `X-max` values. Enable `Reversed` for the X-axis.
   - *Expected:* Plot updates immediately. All customisations are visible.
2. **Persistence Through Plot Type Change:**
   - Change the Plot Type from `Scatter` to `Lines`.
   - *Expected:* The custom title, labels, axis limits, and reversed state are all retained.
3. **Persistence Through Theme Change:**
   - Change the Theme to `theme_ggplot2`.
   - *Expected:* The visual theme updates, but all custom labels and limits remain.
4. **Group Differentiation — Color vs Geometry:**
   - Using a multi-series plot (e.g. `caspl_df_exp` with `x`, `y2`, `y4`), change "Group By" from `Color` to `Geometry` with plot type `Lines`.
   - *Expected:* Series are differentiated by line style rather than colour.
   - Switch plot type to `BarPlot`, keeping "Group By" on `Geometry`.
   - *Expected:* The `Geometry` option is ignored for BarPlot; groups fall back to colour differentiation.
5. **Legend Toggle & Title Persistence:**
   - Uncheck `Show Legend`. Enter a custom legend title.
   - Re-check `Show Legend`.
   - *Expected:* Legend disappears on uncheck; reappears with the previously set title on re-check.
6. **Label Reset on Source Change:**
   - While custom labels are active, switch to a different DataFrame (e.g. from `caspl_df_simple` to `caspl_df_exp`) and replot.
   - *Expected:* Custom labels are **reset to defaults** for the new data source (documented `"source"` reset behaviour).

## Phase 4: File Import (Open Tab)
**Goal:** Verify `CSV` and `XLSX` loading through Extensions and the reading options.

1. **Load Standard CSV:**
   - Go to the **Open** tab.
   - Click **Open File** and select `test/assets/sample_data.csv`.
   - *Expected:* The table populates correctly with the CSV data.
   - **Plot it:** Go to the **Source** tab, ensure `opened file` is selected, choose columns, and click **(Re-)Plot**. Switch to the **Format** tab and apply the `Lines` plot type with `theme_light`.
2. **Load TSV and Delimiter Options:**
   - Go back to the **Open** tab.
   - Click **Open File** and select `test/assets/empty_rows_sample.tsv`.
   - Change `Delimiter` to `Tab` (or ensure `Auto` correctly guesses it). Click **Reload**.
   - *Expected:* The data is correctly parsed into columns.
   - **Plot it:** Go to the **Source** tab, ensure `opened file` is selected, choose columns, and click **(Re-)Plot**. Switch to the **Format** tab, apply the `Scatter` plot type, uncheck `Show Legend`, and change the `X-Label` to "Custom TSV X".
3. **Empty Rows & Non-Standard Headers (XLSX):**
   - Go back to the **Open** tab.
   - Click **Open File** and select `test/assets/empty_rows_top-header_sample.xlsx`.
   - Toggle `Skip empty rows` on and off, clicking **Reload** each time to observe the row count change.
   - Next, click **Open File** and select `test/assets/row2-header_sample.xlsx`. Set `Header row` to `2` and click **Reload**.
   - *Expected:* The table accurately reflects skipped empty rows and uses the 2nd row as column headers.
   - **Plot it:** Go to the **Source** tab, ensure `opened file` is selected, choose columns, and click **(Re-)Plot**. Switch to the **Format** tab and apply the `BarPlot` plot type with `theme_ggplot2`.
4. **Multisheet XLSX & Subheader Skipping:**
   - Go back to the **Open** tab.
   - Click **Open File** and select `test/assets/sample_data-multisheet.xlsx`.
   - Select a different sheet from the dropdown menu.
   - Change `Skip after header` to `1` and click **Reload**.
   - *Expected:* The table updates to show data from the selected sheet, omitting the first data row (subheader).
   - **Plot it:** Go to the **Source** tab, ensure `opened file` is selected, choose columns, and click **(Re-)Plot**. Switch to the **Format** tab, set explicit `X-min` and `X-max` limits, and use `Color` for "Group By".

## Phase 5: Saving & Code Generation
**Goal:** Verify CairoMakie integration, export formats, error handling, and automatic script generation.

1. **Save Plot in Multiple Formats:**
   - Ensure a plot is displayed, then go to the **Save** tab.
   - Enter a filename `my_plot.png` (or use **Select File…** to choose a path) and click **Save Plot**.
   - Change the extension to `.svg` (`my_plot.svg`) and click **Save Plot**.
   - Change the extension to `.pdf` (`my_plot.pdf`) and click **Save Plot**.
   - *Expected:* A success modal appears for each, and the three files are found on disk.
2. **Error Handling — No Plot:**
   - Restart the app and attempt to save before plotting anything.
   - *Expected:* An error modal states "No plot to save."
3. **Error Handling — Invalid Path:**
   - Enter an invalid path (e.g. `/nonexistent_dir/my_plot.png`) in the path field and click **Save Plot**.
   - *Expected:* An error modal catches the failure gracefully without crashing the app.
4. **Overwrite Warning:**
   - Enter `my_plot.png` (a file already saved in step 1) and click **Save Plot**.
   - *Expected:* A confirmation modal asks to overwrite. Click **Overwrite** and verify the file is updated on disk.
5. **Code Generation — Create Script:**
   - With a plot displayed and custom formatting applied, go to the **Save** tab.
   - Enter a path such as `my_plot_script.jl` in the path field.
   - Click **Create Script**.
   - *Expected:* A success modal appears and the `.jl` file is created on disk.
   - Open the generated file and inspect its contents. Run it in a fresh Julia REPL session (without the GUI).
   - *Expected:* The script runs without error and produces a plot visually equivalent to the one shown in the GUI (including plot type, theme, axis labels, and limits).
